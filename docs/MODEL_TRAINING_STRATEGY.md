# AI 모델 학습 및 데이터 전처리 전략 (MODEL_TRAINING_STRATEGY)

본 문서는 스마트 안전 관제 시스템의 인공지능 모델 학습 목표, 전처리 기준, 평가 전략 및 장기 고도화 방향성을 기술합니다. 

---

## 🎯 1. 모델 학습 목표

* **1차 감지 목표:** 실시간 CCTV 영상 채널에서 사람의 관절 움직임을 시퀀스로 입력받아 `Faint(쓰러짐/실신)`와 `Normal(정상 활동)`을 판별하는 이진 분류 모델 학습.
* **최종 감지 목표:** 실제 관제 운영 환경에서 발생할 수 있는 이상행동 이벤트를 신속하고 정밀하게 탐지하여 오탐(False Alarm)과 미탐(Missed Event)을 극복하고 골든타임을 지키는 것.
* **성능 지표 제어:** 
  * **Faint Recall (재현율):** 생명 및 안전에 직결되므로 최우선 순위로 확보(목표치 0.80 이상). 쓰러진 사람을 놓치는 미탐을 최대한 억제합니다.
  * **Faint Precision (정밀도):** 관제원의 무감각화(Alarm Fatigue) 및 현장 확인 비용과 직결되므로 Recall 상승에 따른 오탐 빈도를 일정 수준 이하로 타협 조율합니다.
  * 단순히 오프라인 검증용 Test/Val Accuracy가 높은 모델보다는 실시간 RTSP 중계에서 지연이 적고 바운딩박스/관절 누락에도 안정적으로 수렴하는 모델을 선정합니다.

## 🔄 2. 전체 학습 파이프라인

분석 모델의 전체 학습 및 검증 파이프라인은 다음과 같이 구조화됩니다.

```text
원본 비디오 (.mp4)
 ➡ 비디오 전처리 (30fps 타겟 및 다운샘플링 검토)
 ➡ YOLO26n-pose (Bounding Box 검출 및 17개 관절 키포인트 추출)
 ➡ ByteTrack / SimpleTrackAssigner (동일 인물 track_id 할당)
 ➡ 관절좌표 정규화 및 피처 가공 (Normalized X, Y, Confidence)
 ➡ Per-Track Sequence Buffer 생성 (30프레임 슬라이딩 윈도우)
 ➡ LSTM 입력 데이터 텐서 구성 (Batch, 30, 51)
 ➡ 이진 분류기 학습 (Faint / Normal)
 ➡ Validation & Test 세트 성능 평가
 ➡ Threshold Sweep (임계값 Sweep 분석)
 ➡ Hard Negative 및 미탐 오탐 사례 분석
 ➡ 취약 시나리오 데이터 선별 및 모델 피드백 재학습
```

## ⏱️ 3. FPS 추출 기준 확인

동작 분류(Action Recognition)를 위한 LSTM 모델은 시간의 흐름(FPS)에 극도로 민감합니다. 학습 시점과 런타임 추론 시점의 FPS 차이는 모델이 인식하는 속도(Temporal Scale) 분포를 일그러뜨려 오작동의 주원인이 됩니다.

### 핵심 확인 및 진단 항목

1. **원본 영상 FPS:**
   * 학습 및 테스트에 사용된 공공/자체 수집 데이터셋 원본 영상은 **약 30 FPS** 규격으로 통일되어 있습니다.
2. **전처리 단계에서 추출하는 FPS:**
   * **[확인 필요]** 데이터셋 가공 코드 및 훈련 Sequence 생성기에서 30fps 원본 영상의 모든 프레임을 추출하여 시퀀스 버퍼에 밀어넣고 있는지, 혹은 일정 프레임을 스킵하여 10fps/15fps로 샘플링하여 구축했는지 검증이 요구됩니다.
3. **Sequence 하나를 구성하는 Frame 수 (Sequence Length):**
   * 코드 설정 기본값은 **30 프레임 (Sequence Length = 30)** 입니다.
4. **Sequence 하나가 실제 시간으로 의미하는 길이 (Duration):**
   * 원본 영상 그대로 30fps를 적용해 샘플링 없이 30프레임을 채운다면, 1개의 Sequence는 **정확히 1.0초**의 실제 동작 시간을 내포합니다.
   * 만약 15fps로 다운샘플링하여 30프레임을 구성한다면, 1개의 Sequence는 **2.0초**의 동작 속도를 대변합니다.
5. **학습 시 사용한 FPS와 실시간 추론 시 사용하는 FPS 일치 여부:**
   * **[확인 필요]** 모델 학습 시 사용한 30프레임 데이터의 간격(Time-Interval)과 실시간 RTSP 추론 엔진(`run_rtsp_inference.py`)의 패킷 프레임 수신 속도가 일치해야 합니다. 실시간 엔진이 GPU 성능 문제로 인해 15 FPS로 프레임을 떨어뜨려 분석한다면, 학습 데이터도 15 FPS로 샘플링하여 모델을 다시 훈련해야 합니다.
6. **Frame Skipping 및 Sampling 적용 전략:**
   * 현재 실시간 RTSP 파이프라인은 CPU/GPU 로드 과부하 시 쌓여있는 큐(Queue)를 버리고 최신 프레임을 가져오는 **패킷 드롭 전략**을 기본으로 채택하고 있습니다.
7. **30fps 전체 처리 시 성능 과부하 여부:**
   * 검증 결과 GPU PC 가동 시 4채널 동시 처리에서 약 29.7 FPS 수준을 충족하고 있으나, 다채널 관제망으로 확장할 경우 연산 비용이 지수함수적으로 증가하므로 프레임 레이트 설계 가이드라인을 사전에 조율해야 합니다.

## 📊 4. FPS 설정 방향 가이드

실제 운영 런타임을 고려해 최적의 타협점을 잡기 위한 비교 분석 구조입니다.

| 구분 | 30 FPS 전체 분석 (Full Frame) | 15 FPS 다운샘플링 분석 | 10 FPS 다운샘플링 분석 |
| :--- | :--- | :--- | :--- |
| **장점** | - 신체 움직임 및 쓰러지는 속도 정보 무손실<br>- 프레임 누락에 강건함 | - 연산량 50% 절감<br>- 실시간 다채널 모니터링 부담 대폭 감소 | - CPU/GPU 하드웨어 비용 최소화<br>- 메모리 점유 및 연산 최소화 |
| **단점** | - 연산 자원 대량 소모 (다채널 확장 취약)<br>- 패킷 유실 시 Temporal Gap 발생 | - 빠르게 넘어지는 과도기적인 1~2프레임 포즈 누락 위험 | - 쓰러짐 감지 지연 가능성 존재 (데이터 수집 시간 증가) |
| **권장 방향** | 4채널 이하 소규모 엣지 서버 환경 | **대형 관제망 및 상용 서비스 권장 기본값** | 모니터링 정밀도가 낮은 단순 환경 |

### 💡 최종 모델 FPS 채택 가이드
개발 환경의 로그 분석(`RuntimeMetrics`)을 통해 YOLO의 frame-by-frame latency와 LSTM의 inference latency를 측정하여 합산한 후, **"실시간으로 끊김 없이 큐 적체가 일어나지 않고 돌아갈 수 있는 최대 FPS"**를 1차 기준으로 삼습니다. 이후 해당 타겟 FPS에 일치하도록 학습 데이터를 다운샘플링 가공하여 재훈련한 버전을 프로덕션 모델로 릴리즈합니다.

## 📐 5. 관절좌표 형식 명세

AI 추론 스크립트(`ai/action/classifier.py` 내 `keypoints_to_feature`)는 검출된 17개 관절(Pose Keypoints)을 다음과 같은 정규화 수식으로 피처화하여 LSTM 입력 텐서를 구성합니다.

```python
width, height = infer_frame_size(detection, frame_shape)
# 각 관절(idx) 루프
features.extend([
    float(point.get("x", 0.0)) / max(float(width), 1.0),     # Normalized X (0.0 ~ 1.0)
    float(point.get("y", 0.0)) / max(float(height), 1.0),    # Normalized Y (0.0 ~ 1.0)
    float(point.get("confidence", 0.0))                     # Keypoint Detection Confidence
])
```

### 적용된 좌표 형식 특성 분석

1. **Normalized 좌표 형식 채택:**
   * **사양:** 이미지 해상도의 너비(`width`)와 높이(`height`)를 나눈 0~1 범위의 좌표값입니다.
   * **선택 이유:** CCTV 카메라마다 다를 수 있는 이미지 해상도 편차(Full HD, QHD, HD 등)에 영향받지 않는 일반화 성능을 띱니다.
2. **Bounding Box Relative(인물 기준 상대 정규화) 연동:**
   * **사양:** 비디오 프레임 메타데이터(`frame_shape`)를 구할 수 없는 예외 상황에서는 감지된 인물의 Bbox 가로/세로 길이를 분모로 대체하여 `bbox-relative normalized coordinates`로 적응형 대체 동작을 지원합니다.
   * **선택 이유:** 원거리/근거리 카메라 촬영 구도 차이로 인해 사람의 픽셀 크기가 달라져도, 뼈대 자세 자체의 고유 특징 공간을 모델이 그대로 수렴할 수 있게 만듭니다.
3. **Confidence 스코어 명시:**
   * **사양:** 관절 위치 `(x, y)` 외에 YOLO가 관절을 찾아낸 예측 신뢰도 스코어를 세 번째 채널로 묶어 입력으로 피딩합니다 (17개 관절 * 3채널 = 51차원 피처).
   * **선택 이유:** 가림 현상 등으로 소실된 관절 정보를 예측할 때, 신뢰도가 0.0에 가까운 결측치 상태임을 LSTM에 전달하여 잘못된 자세 판단을 막기 위함입니다.

## 🔄 6. Sequence 구성 및 가공 기준

* **Sequence Length:** 30 프레임 (30 FPS 기준 1.0초 동작)
* **Sequence Stride:** 1 프레임 단위 슬라이딩 윈도우. (매 프레임 분석 결과를 즉시 계산하여 반응성 극대화)
* **이벤트 라벨 부여 정책 (Labeling):**
  * 사람이 바닥에 누워 정지해 있는 "완성된 쓰러짐(Lying)" 구간뿐만 아니라, **서 있는 상태에서 바닥으로 가속하며 고꾸라지는 "과도기적인 쓰러짐 과정"** 프레임 범위에도 Faint(1) 라벨을 주어 신속 감지가 터지도록 설계합니다.
* **클래스 불균형 완화 (Class Balancing):**
  * 훈련 데이터셋 구성 시 정상 활동(Normal) 비디오 비율이 압도적으로 많기 때문에, Faint 발생 구간의 총 시퀀스 수에 맞게 Normal 비퀀스를 무작위 다운샘플링하여 1:1에 근접하도록 클래스 균형을 유지합니다.
* **관절 소실 및 미검출 예외 완화:**
  * **Missing Keypoints:** 프레임 내 신뢰도가 임계값 이하로 떨어진 특정 관절은 EMA 필터(Alpha 0.6)를 이용하여 이전 정상 위치를 외삽(extrapolate)해 보정합니다.
  * **No Person Detected:** 사람이 일시적으로 전혀 잡히지 않는 프레임이 생겨 시퀀스 버퍼에 빈 칸이 생긴 경우, full frame crop 방식을 임시로 가동하여 버퍼 전체가 빈 텐서로 꼬이지 않도록 격리 조치합니다.

## 📈 7. 다차원 평가 매트릭스 수립

모델의 안정성 판정을 위해 다음과 같은 다차원 검증 기준을 충족해야 합니다.

1. **Confusion Matrix 및 오류 세부 분석:**
   * **False Negative (미탐):** 사람이 완전히 누워있는데 정상 활동으로 판별한 최악의 에러 케이스. 원인 분석을 통해 LSTM 레이어 가중치 편향을 교정합니다.
   * **False Positive (오탐):** 바닥에서 짐을 들어올리기 위해 쪼그려 앉는 동작, 혹은 앉아서 작업하는 상태를 쓰러짐으로 판별한 일반적 오탐 케이스. `hard_negative` 분류군으로 저장하여 추가 훈련합니다.
2. **도메인 및 환경별 정밀 평가:**
   * **Indoor Background / Indoor Chromakey / Outdoor:** 조도 변화 및 배경 노이즈가 극도로 심한 실외(Outdoor) 구역 성능을 독립 분리해 검증합니다.
   * **주야간(Day/Night) 및 계절(Season) 편차:** 주간 그림자 노이즈, 야간 저조도에 따른 적외선 화면에서의 YOLO pose 랜드마크 추출 신뢰도 파편화 및 LSTM 오검출률을 필터링합니다.

## 🛠️ 8. 성능 개선 및 실시간 최적화 방향

* **Threshold Sweep 자동화:** 런타임 추론기의 의사결정 신뢰도 임계값(`action-threshold`)을 0.1부터 0.9까지 스위핑하여 Faint Recall과 Precision이 골든 크로스를 이루는 최적의 지점을 검출합니다.
* **Hard Negative Active Learning:** 실제 환경에서 장시간 가동하며 수집된 오경보 영상들에서 포즈 피처를 추출해 정상 활동(Normal) 레이블의 추가 데이터셋으로 가공 및 반복 재학습 루프를 구성합니다.
* **Geometry 편차 제거:** 카메라별 해상도와 시야각(FOV) 차이로 생기는 원근 왜곡을 최소화하기 위해 훈련 데이터 전처리 시 perspective warp 등의 지오메트리 보정을 실험 후보군으로 검토합니다.
